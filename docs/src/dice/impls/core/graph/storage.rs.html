11//! A cache that deals with versions
12//!
13//! This is responsible for performing incremental caching and invalidations
14//! with multiple versions in-flight at the same time.
15//!
16//! The 'VersionedGraph' will track dependency edges and use computed version
17//! number for each cache entry and a global version counter to determine
18//! up-to-date-ness of cache entries.
19//!
20//! TODO(cjhopman): Some of the documentation below indicates intended or future behavior. The intent
21//! is that this documentation describes a "correct" implementation of VersionedGraph as we want it
22//! to be, the implementation is still being updated to match it. This TODO should be removed when
23//! we've matched this behavior.
24//!
25//! Behavior:
26//!
27//! There's two main operations that the storage needs to support:
28//!
29//! - (Op. 1) getting the value of a key (X) at a particular version, this happens one of three ways:
30//!   - (1.1): X has no previous value present: the new value is computed
31//!   - (1.2): X has a previous value present and it is known to be valid at that version: use the existing value
32//!   - (1.3): X has a previous value present and it is invalidated at that version
33//!      - (1.3a): if X's deps' values have not changed since some version where X's stored value was present, reuse the existing value
34//!      - (1.3b): if any of X's deps values have changed, recompute the value
35//! - (Op. 2) processing invalidations being receieved (only at the most recent version)
36//!
37//! To support these operations, nodes store
38//!
39//! - (i) computed values
40//! - (ii) the seriesparalleldeps for that computed value
41//! - (iii) a cellhistory indicating at what versions both (1)+(2) are known to be valid
42//! - (iv) a list of versions where the node is "force-dirtied"
43//! - (v) the non-invalidated most recent reverse dependencies.
44//!
45//! A node may store multiple computed values (and so also multiple (ii) and (iii)) at different versions. Nodes for InjectedKeys, for
46//! example, will store all values that they ever see (as we cannot recompute ones that we drop).
47//!
48//! A node will not know about invalidations outside of its valid cell history.
49//!
50//! For example, consider a scenario with a node A depends on B and this sequence:
51//!
52//! - at v1, A is computed (and so B is as well)
53//! - at v2, B is invalidated, A will also be invalidated
54//! - at v2, B is then computed (but not A)
55//! - at v3, B is invalidated.
56//!
57//! After this sequence, A will not have been informed of the invalidation at v3.
58//! If A is then computed at v2, we will do "deferred dirty propagation" to
59//! inform it of the dirty at v3 (if appropriate).
60//!
61//! For (Op. 2) invalidations: Invalidations can start at both leaf and non-leaf nodes. An invalidation only happens at
62//! the maximum version. Invalidation of a node does roughly:
63//!
64//! ```ignore
65//! // returns a set of keys that also need to be invalidated
66//! invalidate(node, version) -> Vec<Key> {
67//!     if already dirtied? { return vec![] }
68//!     mark_dirty(node, version);
69//!     return take(&mut node.rdeps);
70//! }
71//! ```
72//!
73//! rdeps are stored only for invalidation. Together, this means that a node only needs to stores
74//! rdeps for its latest version and only until being invalidated.
75//!
76//! For (Op. 1) get/compute, there's a couple of non-trivial steps:
77//!
78//! Consider `get(key=K, version=V)`:
79//!
80//! 1. node lookup: will lookup the node for key K, if it's not present, it needs to be computed (so skip (2)),
81//!    if it is present and valid at V it can be directly reused (and we skip everything else, including 4 as
82//!    there's no state to update).
83//!
84//! 2. deps check: this will try to determine if we can reuse the cached (but dirtied) value for a node. First,
85//!    we have the node for key K with a potential value+deps and cell-history H. We will check the latest version
86//!    in H less than V (call this VP). For each dep Dn, we will get it's history at V and check if it also includes
87//!    ther version VP (i.e. Dn had the same value at VP as it does at V). If all deps pass that  check, we can reuse
88//!    K's value from version VP.
89//!
90//! 3. re-compute: the VersionedGraph doesn't care much about this, it's just a normal non-cached compute
91//!
92//! 4. update state: for a value+deps we are storing we have two associated things to make sure are
93//!    up-to-date: (i) the corresponding cell-history and (ii) the rdeps of the node's dependencies.
94//!    (i) has two parts: first determine the initial cell history from the node, there's two cases:
95//!
96//!    a. value is valid from checkdeps: in this case, we'll get a version VP (from (2) above) that indicates
97//!       the version at which we've checked the deps didn't change. we can reuse the cell history for the node
98//!       at version VP.
99//!
100//!    b. value has been recomputed. we can reuse the cell-history for the node if both its value and its deps
101//!       are equal to the new value+deps.
102//!
103//!    If there's no cell-history to reuse (maybe because a newer computation has evicted the associated data), we only
104//!    know that the value+deps are valid at exactly `[V, V+1)` (i.e. just at version V).
105//!    second, we determine the set of versions at which we know the deps have the same value as at the version
106//!    we are computing (V). This is just an intersection of their cell-histories at that version. The valid deps versions
107//!    are further restricted to ensure they don't cross any force-dirtied versions of the node we are computing.
108//!    The final valid cell-history is the union of these two.
109//!    (ii) is easier, we just tell each dep node to record the rdep at the version we are computing. If the rdep has already seen a
110//!    dirty at a later version, it does not need to record the rdep (and our computing of (i) will reflect that dirty).
111//!
112//! "forced-dirty":
113//! The non-leaf nodes that are directly invalidated are going to be marked as "force-dirtied" at that
114//! version, while rdeps of those just get marked as invalidated. If a dep is marked as "force-dirtied"
115//! at a version, we will never reuse its value across that version based on its deps not changing.
116//!
117//! This means that we need to store these markers forever and that a deps check cannot cross these markers
118//! and that when computing a cell-history after recomputing we must ensure that the deps-based part of that
119//! history does not cross the markers (it's okay for value-based equality to make the history cross those
120//! values).
121//!
122//! Code structure:
123//!
124//! This is structured so that VersionedGraphNode is mostly responsible for the parts of these operations
125//! that are specific to a specific node and that VersionedGraph is responsible for the rest (so things
126//! crossing multiple nodes, managing the map of nodes, dealing with nodes not yet being in the map, etc.).
127//!
128//! Potential improvements:
129//!
130//! 1. We could decouple the cell history for (i) and (ii). Right now, if
131//!    the list of deps changes but the computed value stays the same, we don't record that dependents could
132//!    reuse the value across the two versions.
133//!
134//! 2. For the deps check, instead of checking only the against the most recent previous version, we could
135//!    check against the entire cell-history for our potential reused value. The idea of the deps check is that
136//!    if all of the deps are in a matching state for any version within our cell-history, we can reuse that
137//!    cell. We currently check only against the most recent version because it is significantly simpler and
138//!    in practice is almost as good as checking against the whole history.
139//!
140//! FAQ:
141//!
142//! Q: CellHistory is complex, couldn't we just operate on single VersionRange?
143//!
144//! A: Consider the case where you have key A depends on B depends on C, and the following sequence of operations:
145//!
146//! - At V1, compute A (and so B and C).
147//! - Change C to a new value (computation is now at V2).
148//! - At V2, compute C.
149//!   - A and B will now be dirty at V2
150//! - Change C back to its initial (V1) value (computation is now at V3)
151//! - At V3, compute A
152//!
153//! When B is recomputed at V3, we will see a history like: `[[V1, V2), [V3, inf)]` and see that we can reuse
154//! the A that was computed at V1. If nodes only stored a valid VersionRange, B would have lost the information
155//! that it has the same value at V1 and V3.
156//!
157//! Q: Why use CellHistory-based dependency checks at all? Could we store a node's deps seen values along with the
158//! deps and do value-based dependency checks?
159//!
160//! A: This could be an interesting avenue to explore. It has several potential drawbacks: (1) significantly increased
161//! memory use to store pointers (probably Arc) to the values in each dependent (2) value equality is potentially
162//! expensive and this approach may require additional caching to avoid that cost (for example, at least a
163//! transaction-level cache of such comparisons, otherwise we'd need to do O(E) equality checks).
164//!
165//! Q: Could we do strong-hash-based equality checks instead?
166//!
167//! A: This could also be interesting to explore. It's possible that this could resolve all the issues with doing
168//! value-based dep checks.

194/// The actual incremental cache that checks versions and dependency's versions
195/// to maintain correct caching based on versions and the versions of its
196/// dependencies.
197#[derive(Allocative)]
198pub(crate) struct VersionedGraph {
199    /// storage that stores every version forever
200    /// This storage is implemented so that the map keys are composed of the versions for which
201    /// the node changes. Corresponding to each key is a node storing the values and the history.
202    /// VacantGraphEntries can only be present when no other entries are present for the key at
203    /// any version.
204    pub(crate) nodes: HashMap<DiceKey, VersionedGraphNode>,
205}
206
207impl VersionedGraph {
208    pub(crate) fn new() -> Self {
209        Self {
210            nodes: Default::default(),
211        }
212    }
213
214    /// Gets the entry corresponding to the cache entry if up to date.
215    pub(crate) fn get(&self, key: VersionedGraphKey) -> VersionedGraphResult {
216        if let Some(entry) = self.nodes.get(&key.k) {
217            entry.at_version(key.v)
218        } else {
219            VersionedGraphResult::Compute
220        }
221    }
222
223    /// updates the cached value based on the given key and versions. The value
224    /// is only stored if the version is newer than what is stored.
225    /// Returns the new entry, and an optional old entry that was invalidated due to this update
226    #[cfg_attr(debug_assertions, instrument(level = "debug", skip(self, value, storage_type, deps, reusable), fields(key = ?key)))]
227    pub(crate) fn update(
228        &mut self,
229        key: VersionedGraphKey,
230        value: DiceValidValue,
231        reusable: ValueReusable,
232        deps: Arc<SeriesParallelDeps>,
233        storage_type: StorageType,
234        invalidation_paths: TrackedInvalidationPaths,
235    ) -> (DiceComputedValue, bool) {
236        if let StorageType::Injected = storage_type {
237            unreachable!(
238                "Injected keys should not receive update calls, as those are only from a compute() finishing and InjectedKeys have no compute()"
239            );
240        };
241
242        let mut valid_deps_versions = VersionRange::begins_with(VersionNumber::ZERO).into_ranges();
243
244        // Add rdeps.
245        for dep in deps.iter_keys() {
246            let node = self.nodes.get_mut(&dep).expect("dependency should exist");
247            node.add_rdep_at(key.v, key.k);
248            node.intersect_valid_versions_at(key.v, &mut valid_deps_versions);
249        }
250
251        let invalidation_paths = invalidation_paths.for_dependent(key.k);
252
253        // Update entry.
254        match self.nodes.get_mut(&key.k) {
255            Some(entry) => entry.on_computed(
256                key,
257                value,
258                valid_deps_versions,
259                reusable,
260                deps,
261                invalidation_paths,
262            ),
263
264            None => (
265                self.update_empty(
266                    key.k,
267                    key.v,
268                    value,
269                    valid_deps_versions,
270                    deps,
271                    invalidation_paths,
272                ),
273                true,
274            ),
275        }
276    }
277
278    /// Invalidates an entry and its transitive rdeps. Returning true if this caused any type of
279    /// change
280    pub(crate) fn invalidate(
281        &mut self,
282        key: VersionedGraphKey,
283        invalidate: InvalidateKind,
284        invalidation_priority: InvalidationSourcePriority,
285    ) -> bool {
286        let entry = match self.nodes.get_mut(&key.k) {
287            Some(entry) => entry,
288            _ => {
289                let new_entry = match invalidate {
290                    InvalidateKind::Update(value, StorageType::Injected) => {
291                        VersionedGraphNode::Injected(InjectedGraphNode::new(
292                            key.k,
293                            key.v,
294                            value,
295                            invalidation_priority,
296                        ))
297                    }
298                    InvalidateKind::Update(value, StorageType::Normal) => {
299                        VersionedGraphNode::Occupied(OccupiedGraphNode::new(
300                            key.k,
301                            value,
302                            Arc::new(SeriesParallelDeps::None),
303                            VersionRange::begins_with(key.v).into_ranges(),
304                            ForceDirtyHistory::new(),
305                            TrackedInvalidationPaths::new(invalidation_priority, key.k, key.v),
306                        ))
307                    }
308                    _ => {
309                        let mut node = VacantGraphNode::new(key.k, invalidation_priority);
310                        // invalidated and force_dirty for a vacant node are going to have the same behavior in practice.
311                        node.force_dirty(key.v, invalidation_priority);
312                        VersionedGraphNode::Vacant(node)
313                    }
314                };
315
316                self.nodes.insert(key.k, new_entry);
317                return true;
318            }
319        };
320
321        let queue = {
322            let res = match invalidate {
323                InvalidateKind::Invalidate => {
324                    entry.mark_invalidated(key.v, Some(invalidation_priority))
325                }
326                InvalidateKind::ForceDirty => entry.force_dirty(key.v, invalidation_priority),
327                InvalidateKind::Update(value, _) => {
328                    entry.on_injected(key.v, value, invalidation_priority)
329                }
330            };
331
332            if let InvalidateResult::Changed(rdeps) = res {
333                rdeps.into_iter().flatten().collect()
334            } else {
335                return false;
336            }
337        };
338
339        self.invalidate_rdeps(key.v, queue);
340        true
341    }
342

344    // ------------------------- Implementation functions below --------------------
345    // -----------------------------------------------------------------------------
346
347    #[cfg_attr(debug_assertions, instrument(level = "debug", skip(self, value, deps), fields(key = ?key, v = %v, valid_deps_versions = ?valid_deps_versions)))]
348    fn update_empty(
349        &mut self,
350        key: DiceKey,
351        v: VersionNumber,
352        value: DiceValidValue,
353        mut valid_deps_versions: VersionRanges,
354        deps: Arc<SeriesParallelDeps>,
355        invalidation_paths: TrackedInvalidationPaths,
356    ) -> DiceComputedValue {
357        debug!("making new graph entry because previously empty");
358        valid_deps_versions.insert(VersionRange::bounded(v, VersionNumber::new(v.0 + 1)));
359        let entry = OccupiedGraphNode::new(
360            key,
361            value,
362            deps,
363            valid_deps_versions,
364            ForceDirtyHistory::new(),
365            invalidation_paths,
366        );
367
368        let res = entry.computed_val(v);
369        self.nodes.insert(key, VersionedGraphNode::Occupied(entry));
370        res
371    }
372
373    fn invalidate_rdeps(&mut self, version: VersionNumber, mut queued: HashSet<DiceKey>) {
374        let mut queue: Vec<_> = queued.iter().copied().collect();
375
376        while let Some(rdep) = queue.pop() {
377            if let Some(node) = self.nodes.get_mut(&rdep) {
378                if let InvalidateResult::Changed(Some(rdeps)) = node.mark_invalidated(version, None)
379                {
380                    for dep in rdeps.into_iter() {
381                        if queued.insert(dep) {
382                            queue.push(dep);
383                        }
384                    }
385                }
386            }
387        }
388    }
389}

391pub(crate) enum ValueReusable {
392    /// Directly compare the values for equality to determine if the node can be reused
393    EqualityBased,
394    /// Compare the value's version history to determine if the node can be reused
395    VersionBased(VersionNumber),
396}
397
398impl ValueReusable {
399    pub(crate) fn is_reusable(
400        &self,
401        new_value: &DiceValidValue,
402        new_deps: &SeriesParallelDeps,
403        value: &OccupiedGraphNode,
404    ) -> bool {
405        match self {
406            ValueReusable::EqualityBased => {
407                if new_deps != &***value.deps() {
408                    return false;
409                }
410                new_value.equality(value.val())
411            }
412            // For version-based, the deps are guaranteed to match if `version` is in the node's verified versions.
413            ValueReusable::VersionBased(version) => value.is_verified_at(*version),
414        }
415    }
416}
