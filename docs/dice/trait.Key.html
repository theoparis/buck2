Expand description

The computation Key that maps to a value. The key will be used as an index for caching the computed values.

§Cloning

The Key will need to be cloned. This occurs whenever the key is recomputed and needs to be stored. During normal lookups, only a reference is needed.

The result also needs to be cloned whenever it is requested from DICE. Cloning therefore should be cheap.

fn equality(x: &Self::Value, y: &Self::Value) -> bool

If computed value is equal to previously cached value, DICE won’t invalidate graph nodes depending on this node.

It is safe to return false from this function when values are equal, but returning true when values are not equal would result in inconsistent graph state.

fn validity(_x: &Self::Value) -> bool

If the computed value is false, DICE will consider that result to be a transient value that won’t be re-used on subsequent computations. It will, however, reuse that value for all on-going computations at the current version.

The default here is true, but computations should override this if its expected that they may occasionally produce transient values.
