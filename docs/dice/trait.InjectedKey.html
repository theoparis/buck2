Expand description

Specialized version of Key above. This type of Key is never computed. It should always be injected onto the graph before being requested via DiceCtx::changed_to(Key, Value). Therefore, the compute function on this Key is never called.

TODO when rust gets smarter about negative trait bounds and specializations, we can make injected and normal key types disjoint, and not have this clone issue nor require a panic! implementation in compute function, both of which are horrible and breaks semantics of traits.
